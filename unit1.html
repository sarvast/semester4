Computer Graphics & Animation (BCA-4001)

1. Introduction to Interactive Computer Graphics PYQ 2022, 2024

Interactive Computer Graphics: Involves two-way communication between user and computer. The user controls the content, structure, and appearance of objects and images by using input devices (keyboard, mouse, touch). The image is updated on the display in real-time response to user inputs.

Passive (Non-Interactive) Graphics: The user has no control over the image. Example: Watching TV or a standard screensaver.

Conceptual Framework:

  • Application Model: Represents the data or objects to be displayed.
  • Application Program: Creates and manipulates the model based on user input.
  • Graphics System: Intermediary between the application and the display hardware.

2. Advantages of Interactive Graphics

  • Higher Quality: Produces precise and high-resolution images.
  • Better User Interface: Makes human-computer interaction more natural (GUIs).
  • Simulation: Allows testing of designs and scenarios without physical prototypes (CAD/Flight Simulators).
  • Visualization: Helps visualize complex data (graphs, charts, 3D models).
  • Productivity: Increases design speed and accuracy in engineering and creative fields.

3. Classification of Applications

Computer graphics applications can be classified into:

  1. Business Graphics: Bar charts, pie charts for analyzing financial/statistical data.
  2. CAD (Computer-Aided Design): Engineering drawings, architectural plans. PYQ 2022
  3. Entertainment: Movies, games, animation.
  4. Education & Training: Models for teaching, flight simulators.
  5. Image Processing: Modifying existing photos/images (Photoshop, Medical Imaging like CT Scans).

4. Hardware & Software Development

Hardware:

  • Input Devices: Mouse, Keyboard, Tablet, Light Pen, Joystick.
  • Output Devices (Display Technologies):
    • CRT (Cathode Ray Tube): Uses electron guns to fire electrons at a phosphorescent screen.
      • Random Scan (Vector): Electron beam directed only to parts of screen where picture is drawn. High resolution, no aliasing.
      • Raster Scan: Electron beam scans across the screen row by row from top to bottom. Universally used in TVs/Monitors.
    • Flat Panel Displays: Portable and thinner.
      • LCD (Liquid Crystal Display): Uses liquid crystals to block/allow light.
      • LED / Plasma: Emissive displays with better contrast.
  • Video Memory (Frame Buffer): Stores the pixel data (color/intensity) to be displayed. Depth (bits per pixel) determines color range.
  • Graphics Processor (GPU): Dedicated chip for rendering images quickly, offloading work from CPU.

Scan Conversion Algorithms: Important

  • DDA (Digital Differential Analyzer): Uses floating-point arithmetic. Slower. Cumulative error build-up.
  • Bresenham’s Line Algorithm: Uses integer arithmetic (addition/subtraction). Faster and more accurate.
  • Circle Generation: Midpoint Circle Algorithm (also uses integer arithmetic).

Software:

  • GKS (Graphics Kernel System): First international standard (ISO) for 2D graphics.
  • PHIGS: Programmers' Hierarchical Interactive Graphics System (for 3D modeling).
  • OpenGL / DirectX / Vulkan: Modern APIs for hardware-accelerated rendering.

5. Pixel, Resolution, Aspect Ratio PYQ 2022, 2023

Pixel (Picture Element): The smallest addressable screen element.

Resolution: Number of pixels (Width × Height). Higher resolution = sharper image.

Aspect Ratio: Ratio of image Width to Height (e.g., 4:3, 16:9). PYQ 2022, 2023

Database Management System (BCA-4002)

1. Introduction to DBMS

Database: A clear, organized collection of data.
DBMS: Software to define, construct, manipulate, and share databases.

Characteristics of Database Approach: PYQ 2024

  • Self-Describing Nature: Contains data + metadata (catalog).
  • Insulation between Programs and Data: Data Abstraction.
  • Support of Multiple Views: Different users see different parts of data.
  • Data Sharing & Multi-user Transaction Processing: Concurrent access.

2. Data Models

A collection of concepts for describing data, relationships, and constraints.

  • High-Level (Conceptual): Close to user view. E.g., Entity-Relationship (ER) Model.
  • Representational (Implementation): Between high and low level. E.g., Relational Model (Tables), Hierarchical (Tree), Network (Graph). PYQ 2024
  • Low-Level (Physical): How data is stored on disk.

3. DBMS Architecture PYQ 2024

Three-Schema Architecture (ANSI/SPARC):

  1. Internal/Physical Level: Physical storage details. PYQ 2024-25
  2. Conceptual/Logical Level: Structure of whole database (Tables, Constraints). Hides physical details. PYQ 2022
  3. External/View Level: User-specific views. (e.g., Student sees only grades, Admin sees everything).

Client-Server Architectures:

  • 2-Tier: Client Application ↔ Database Server (Direct).
  • 3-Tier: Client ↔ Application Server ↔ Database Server (Scalable, Secure).

4. Data Independence PYQ 2024-25

The capacity to change the schema at one level without changing the schema at the next higher level.

  • Logical Data Independence: Modify Conceptual Schema without changing External Views/Application Programs. (e.g., Adding a new column doesn't break app code).
  • Physical Data Independence: Modify Internal Schema without changing Conceptual Schema. (e.g., Changing from HDD to SSD or changing file indexing structure).

5. DBMS Languages & Users

  • DDL (Data Definition Language): Define structure (CREATE, ALTER, DROP, TRUNCATE). MCQ Topic
  • DML (Data Manipulation Language): Manage data (SELECT, INSERT, UPDATE, DELETE).
  • DCL (Data Control Language): Control access (GRANT, REVOKE).
  • TCL (Transaction Control Language): Manage transactions (COMMIT, ROLLBACK).

Database Users: DBA (Admin), Application Programmers, Sophisticated Users (Analysts), Naive Users (End-users).

Software Engineering (BCA-4003)

1. Definition and Basics

Software Engineering (IEEE): "The application of a systematic, disciplined, quantifiable approach to the development, operation, and maintenance of software." PYQ 2024

Father of SE: Watts S. Humphrey PYQ 2024

Software Characteristics: PYQ 2022

  • Logic-based, not physical (does not wear out).
  • Engineered/Developed, not manufactured.
  • Custom-built usually, unlike hardware components.
  • Complex and invisible.

2. Generic View of Software Engineering

Software Engineering is a Layered Technology:

  • Tools: Automated support (CASE tools).
  • Methods: Technical "how-to" for building software (Analysis, Design, Coding, Testing).
  • Process: The glue that holds layers together; defines the framework (Activities, Actions, Tasks).
  • Quality Focus: The foundation; bedrock that supports software engineering.

Process Framework Activities: Communication, Planning, Modeling, Construction, Deployment.

3. Software Paradigms (SDLC Models) PYQ 2022, 2024

Waterfall Model (Linear Sequential): Classic Lifecycle. PYQ 2024
Req → Design → Coding → Testing → Maintenance. Simple but rigid.

Incremental Model: Delivers software in small increments/versions.

RAD (Rapid Application Development): High-speed adaptation of linear model using component-based construction.

Spiral Model (Evolutionary): Risk-driven. Couples iterative nature of prototyping with systematic aspects of waterfall.
Phases: Planning, Risk Analysis, Engineering, Evaluation. Best for high-risk, large projects. PYQ 2022

Prototype Model: Building a mock-up to clarify requirements early.
Use when: Requirements are unclear. User gets a "feel" of the system before actual development.

Agile Model: Iterative, incremental, flexible. Focus on customer satisfaction and responding to change (e.g., Scrum, XP).

Optimization Techniques (BCA-4004)

1. Linear Programming Problem (LPP)

Central Problem: Optimization (Maximizing Profit or Minimizing Cost) subject to limited resources (constraints). All relationships are linear.

Structure: Max/Min $Z = c_1x_1 + ... + c_nx_n$ (Objective)
Subject to: $a_{11}x_1 + ... \leq b_1$ (Constraints)
and $x_i \geq 0$ (Non-negativity).

Definitions: PYQ 2022

  • Solution: Any set of variables satisfying constraints.
  • Feasible Solution: A solution that also satisfies non-negativity restrictions ($x \geq 0$).
  • Optimal Solution: A feasible solution that optimizes $Z$.
  • Convex Set: A set of points where the line segment joining any two points lies entirely within the set. (Feasible region is always convex).

2. Graphical Method PYQ 2022, 2024

Visual method for problems with 2 variables.

  • Plot constraint lines. Use ($=0$) to find intercepts.
  • Identify Feasible Region (common shaded area).
  • Corner Point Theorem: Optimal solution always lies at one of the corner points (vertices) of the feasible region.

3. Simplex Method

Iterative algebraic procedure for solving LPP with more than 2 variables.

  • Slack Variable ($S \ge 0$): Added to $\leq$ constraint (e.g., $2x + y \le 10 \to 2x + y + S = 10$). Represents unused resource. PYQ 2024
  • Surplus Variable ($-S$): Subtracted from $\geq$ constraint. Represents excess amount over requirement.
  • Artificial Variable ($A \ge 0$): Added to $\geq$ or $=$ constraints to get initial identity basis matrix for starting the Simplex table. Has no physical meaning (Penalty M method used to remove it).
  • Simplex Method: Iterative process to find optimal solution.
    • Slack Variable ($S$): Added to $\le$ constraint (unused resources). Cost coeff = 0.
    • Surplus Variable ($-S$): Subtracted from $\ge$ constraint (excess). Cost coeff = 0.
    • Artificial Variable ($A$): Added to $\ge$ or $=$ constraint to get initial basis. Cost coeff = $-M$ (Big M Method).
    • Optimality Condition: For Maximization, stop when all $C_j - Z_j \le 0$ (for non-basic variables).

4. Primal and Dual

Every LPP (Primal) has an associated LPP called Dual.

  • Max Z in Primal $\leftrightarrow$ Min W in Dual.
  • No. of constraints in Primal = No. of variables in Dual.
  • Dual of Dual is Primal. PYQ 2023

5. Transportation & Assignment

Transportation Problem: Minimizing cost of shipping goods from $m$ sources to $n$ destinations.
Methods: North West Corner (NWC), Least Cost, VAM. PYQ 2022

Assignment Problem: Assigning $n$ jobs to $n$ machines/people 1-to-1 to minimize cost/time.
Method: Hungarian Method.

Mathematics-III (BCA-4005)

1. Complex Number System

Definition: A number of the form $z = x + iy$, where $x, y \in \mathbb{R}$ and $i = \sqrt{-1}$.

  • Real Part: $Re(z) = x$
  • Imaginary Part: $Im(z) = y$
  • Conjugate ($\bar{z}$): If $z = x + iy$, then $\bar{z} = x - iy$.
  • Modulus ($|z|$): Distance from origin. $|z| = \sqrt{x^2 + y^2}$. PYQ 2022
  • Powers of $i$: Cycle of 4 ($i, -1, -i, 1$).
    Example: $i^{19} = i^{4(4)+3} = i^3 = -i$ PYQ 2024

2. Polar Form & Arguments

Representing $z$ using radius $r$ and angle $\theta$.

Form: $z = r(\cos\theta + i\sin\theta)$ or $z = re^{i\theta}$ (Euler's Form).

  • $r = |z|$ (Modulus)
  • $\theta = \tan^{-1}(y/x)$ (Argument or Amplitude).
  • Principal Argument: Value of $\theta$ in $(-\pi, \pi]$.

3. De Moivre's Theorem PYQ 2023

Theorem: For any real number $n$, $(\cos\theta + i\sin\theta)^n = \cos(n\theta) + i\sin(n\theta)$.

Use: Finding powers and roots of complex numbers.

Roots of Unity: Solutions to $z^n = 1$.
Cube roots of unity ($1, \omega, \omega^2$): sum $1+\omega+\omega^2=0$, product $\omega^3=1$. PYQ 2023

  • De Moivre's Theorem: $(\cos\theta + i\sin\theta)^n = \cos(n\theta) + i\sin(n\theta)$.
  • Roots of Unity ($1^{1/3}$): The cube roots of unity are $1, \omega, \omega^2$.
    • Properties: $1 + \omega + \omega^2 = 0$ and $\omega^3 = 1$.
  • Analytic Functions: A function $f(z)$ differentiable at every point in a domain.
    • Cauchy-Riemann (C-R) Equations: Necessary condition for differentiability.
      $\frac{\partial u}{\partial x} = \frac{\partial v}{\partial y}$ and $\frac{\partial u}{\partial y} = -\frac{\partial v}{\partial x}$.

4. Functions of Complex Variable

Function $f(z)$: A rule assigning a complex number $w$ to $z$. $w = f(z) = u(x,y) + iv(x,y)$.

  • Limit: $lim_{z \to z_0} f(z) = L$. Path independent approach required.
  • Continuity: Limit exists and equals function value.
  • Differentiability: Derivative exists. A function differentiable everywhere is Analytic (Holomorphic). Verified using Cauchy-Riemann Equations ($u_x=v_y, u_y=-v_x$). PYQ 2024
