Topic-wise Notes

Computer Graphics & Animation

Curves & Surfaces

  • Polygon Meshes: Collection of vertices, edges, and polygons representing 3D surfaces.
  • Parametric Representation: Surfaces defined by parameters $(u, v)$ instead of $y=f(x)$, allowing complex shapes.
  • Interpolation vs. Approximation:
    • Interpolation: Curve passes through all control points.
    • Approximation: Curve is influenced by control points but doesn't necessarily pass through them (e.g., Bezier, B-Spline).
  • Bezier Curve:
    • Defined by $n+1$ control points. Degree of curve is $n$.
    • Properties:
      • Endpoint Interpolation: Passes through first ($P_0$) and last ($P_n$) points.
      • Convex Hull: The curve lies completely within the convex hull of its control points.
      • Global Control: Moving any single control point changes the entire shape of the curve.
  • B-Spline Curve:
    • Generalization of Bezier curves to overcome its limitations.
    • Properties:
      • Local Control: Moving a control point only affects a specific segment of the curve.
      • Degree Independence: The degree of the curve is independent of the number of control points.
  • Continuity:
    • $G^0$: Curves touch (Position continuity).
    • $G^1$: Same tangent direction (Slope continuity).
    • $C^1$: Same tangent direction and magnitude (Parametric continuity).
  • Comparison:
    • Bezier: Simple, Global control, Degree depends on points ($n$).
    • B-Spline: Complex, Local control, Degree independent of points, can represent conics.
  • 3D Projections: Converting 3D objects to 2D screen.
    • Parallel Projection: Preserves relative proportions (e.g., Orthographic). Good for engineering.
    • Perspective Projection: Objects farther away appear smaller. Realistic. Parallel lines meet at a Vanishing Point.
  • Hidden Surface Removal: Determining visible surfaces.
    • Back-Face Culling: Hides polygons facing away.
    • Z-Buffer (Depth Buffer): Pixel-based depth comparison.

Database Management System (DBMS)

Relational Model & SQL

  • Relational Model: Data in tables (Relations). Row=Tuple, Column=Attribute.
  • Integrity Constraints: Entity (PK $\neq$ Null), Referential (FK matches PK), Domain (Valid values).
  • Relational Algebra:
    • Selection ($\sigma$): Horizontal subset (Rows).
    • Projection ($\pi$): Vertical subset (Columns).
    • Difference ($-$): Rows in A but not B.
    • Cartesian Product ($\times$): All combinations of rows.
  • SQL (Structured Query Language):
    • DDL: CREATE, ALTER, DROP.
    • DML: SELECT, INSERT, UPDATE, DELETE.
    • Keywords: DISTINCT (Remove duplicates), WHERE (Filter).

Transaction Management & Concurrency

  • Transaction: logical unit of work (e.g., Transfer money).
  • ACID Properties:
    • Atomicity: All or nothing. (Commit/Rollback).
    • Consistency: DB remains consistent before/after transaction.
    • Isolation: Transactions don't interfere with each other.
    • Durability: Committed changes are permanent (survive failure).
  • Concurrency Control: Managing simultaneous access.
    • Locks: Binary, Shared/Exclusive.
    • Deadlock: Two transactions waiting for each other.

Software Engineering

Implementation & Coding Standards

  • Implementation: Translating design into source code.
  • Good Coding Style: Focuses on readability, maintainability, and modularity.
  • Structured Programming: Uses sequence, selection, and iteration. Avoids `GOTO`.
  • Validation vs. Verification:
    • Validation: "Are we building the right product?" (User needs).
    • Verification: "Are we building the product right?" (Specs).
  • Software Testing Strategies:
    • White Box (Glass Box): Testing internal logic/code (e.g., Basis Path). Developer does this.
    • Black Box: Testing functionality (Input/Output) without knowing code (e.g., Boundary Value Analysis).
    • Levels: Unit (Individual modules), Integration (Interaction), System (Complete).
  • Software Configuration Management (SCM): Managing changes (Version Control) and controlling software evolution.
  • Testing Types:
    • Alpha Testing: By internal team in developer environment.
    • Beta Testing: By real users in real environment.
    • Regression Testing: Re-testing after modification to ensure no new bugs.
  • Software Maintenance:
    • Corrective: Fixing bugs.
    • Adaptive: Moving to new platform/OS.
    • Perfective: Adding new features.
    • Preventive: Future-proofing (Refactoring).

Optimization Techniques

Project Management (PERT & CPM)

  • Network Diagram: Visualizes project activities (Arrows $\rightarrow$) and events (Nodes $\circ$).
  • CPM (Critical Path Method): Deterministic (Fixed times). Finds the longest path (Critical Path). Zero float activities are critical.
  • PERT (Program Evaluation Review Technique): Probabilistic (Uncertain times). Used for R&D.
  • Time Estimates (PERT): Optimistic ($t_o$), Most Likely ($t_m$), Pessimistic ($t_p$).
  • Expected Time: $t_e = \frac{t_o + 4t_m + t_p}{6}$.
  • Float (Slack): Spare time available.
    • Total Float: Max delay without delaying project finish. ($LS - ES$ or $LF - EF$).
    • Free Float: Delay without delaying successor start.
    • Independent Float: Delay without affecting predecessor or successor.

Mathematics-III

Ordinary Differential Equations (First Order)

  • Order vs. Degree: Order = Highest derivative. Degree = Power of highest derivative.
  • Variable Separable: Separate $x$ and $y$ terms and integrate.
  • Linear ODE: Form $\frac{dy}{dx} + Py = Q$.
    • Integrating Factor (IF) = $e^{\int P dx}$. Solution: $y(IF) = \int (Q \cdot IF) dx + C$.
  • Homogeneous Equation: Degree of terms in $x$ and $y$ is same. Put $y=vx$.
  • Bernoulli’s Equation: form $\frac{dy}{dx} + Py = Qy^n$. Solution: Divide by $y^n$, put $v = y^{1-n}$ to make it Linear.
  • Exact Differential Equation: $Mdx + Ndy = 0$ is exact if $\frac{\partial M}{\partial y} = \frac{\partial N}{\partial x}$.
    Solution: $\int M dx (\text{y const}) + \int (\text{terms of N free from x}) dy = C$.
  • Orthogonal Trajectories: Curves cutting a family of curves at $90^\circ$.
    Method: Form differential equation. Replace $\frac{dy}{dx}$ with $-\frac{dx}{dy}$. Solve new DE.
