Unit 1 Content

Computer Graphics & Animation (BCA-4001)

1. Introduction to Interactive Computer Graphics PYQ 2022, 2024

Interactive Computer Graphics: Involves two-way communication between user and computer. The user controls the content, structure, and appearance of objects and images by using input devices (keyboard, mouse, touch). The image is updated on the display in real-time response to user inputs.

Passive (Non-Interactive) Graphics: The user has no control over the image. Example: Watching TV or a standard screensaver.

Conceptual Framework:

  • Application Model: Represents the data or objects to be displayed.
  • Application Program: Creates and manipulates the model based on user input.
  • Graphics System: Intermediary between the application and the display hardware.

2. Advantages of Interactive Graphics

  • Higher Quality: Produces precise and high-resolution images.
  • Better User Interface: Makes human-computer interaction more natural (GUIs).
  • Simulation: Allows testing of designs and scenarios without physical prototypes (CAD/Flight Simulators).
  • Visualization: Helps visualize complex data (graphs, charts, 3D models).
  • Productivity: Increases design speed and accuracy in engineering and creative fields.

3. Classification of Applications

Computer graphics applications can be classified into:

  1. Business Graphics: Bar charts, pie charts for analyzing financial/statistical data.
  2. CAD (Computer-Aided Design): Engineering drawings, architectural plans. PYQ 2022
  3. Entertainment: Movies, games, animation.
  4. Education & Training: Models for teaching, flight simulators.
  5. Image Processing: Modifying existing photos/images (Photoshop, Medical Imaging like CT Scans).

4. Hardware & Software Development

Hardware:

  • Input Devices: Mouse, Keyboard, Tablet, Light Pen, Joystick.
  • Output Devices (Display Technologies):
    • CRT (Cathode Ray Tube): Uses electron guns to fire electrons at a phosphorescent screen.
      • Random Scan (Vector): Electron beam directed only to parts of screen where picture is drawn. High resolution, no aliasing.
      • Raster Scan: Electron beam scans across the screen row by row from top to bottom. Universally used in TVs/Monitors.
    • Flat Panel Displays: Portable and thinner.
      • LCD (Liquid Crystal Display): Uses liquid crystals to block/allow light.
      • LED / Plasma: Emissive displays with better contrast.
    • CRT Variants:
      • Direct View Storage Tube (DVST): Stores picture information as a charge distribution behind the phosphor-coated screen. No refresh needed. Issues: No partial erasure, no animation.
      • Color CRT (Shadow Mask): Uses 3 electron guns (RGB) and a shadow mask (metal plate with holes) to light up specific phosphor dots.
  • Video Memory (Frame Buffer): Stores the pixel data (color/intensity) to be displayed. Depth (bits per pixel) determines color range.
  • Graphics Processor (GPU): Dedicated chip for rendering images quickly, offloading work from CPU.

Scan Conversion Algorithms: Important

  • DDA (Digital Differential Analyzer): Uses floating-point arithmetic. Slower. Cumulative error build-up.
  • Bresenham’s Line Algorithm: Uses integer arithmetic (addition/subtraction). Faster and more accurate.
  • Circle Generation: Midpoint Circle Algorithm (also uses integer arithmetic).

Software:

  • GKS (Graphics Kernel System): First international standard (ISO) for 2D graphics.
  • PHIGS: Programmers' Hierarchical Interactive Graphics System (for 3D modeling).
  • OpenGL / DirectX / Vulkan: Modern APIs for hardware-accelerated rendering.

5. Pixel, Resolution, Aspect Ratio PYQ 2022, 2023

Pixel (Picture Element): The smallest addressable screen element.

Resolution: Number of pixels (Width × Height). Higher resolution = sharper image.

Aspect Ratio: Ratio of image Width to Height (e.g., 4:3, 16:9). PYQ 2022, 2023

Scan Converting Lines

  • Definition: The process of converting geometric primitives (like lines) into pixel coordinates on a raster grid is called scan conversion or rasterization.
  • Digital Differential Analyzer (DDA) Algorithm:
    • A line drawing algorithm based on the equation $y = mx + c$.
    • It calculates the next point by adding a step value ($\Delta x$ or $\Delta y$) to the current coordinate.
    • Disadvantage: It involves floating-point calculations (rounding off), which makes it slower than integer-based algorithms.
  • Bresenham’s Line Algorithm:
    • An accurate and efficient raster line-generating algorithm that uses only integer addition and subtraction.
    • It uses a Decision Parameter ($p_k$) to determine whether to choose the next pixel as $(x_k+1, y_k)$ or $(x_k+1, y_k+1)$.
    • If $p_k < 0$, the next point is $(x_{k+1}, y_k)$; otherwise, it is $(x_{k+1}, y_{k+1})$ (for slope $0 < m < 1$).

Scan Converting Circles & Clipping

  • Symmetry: A circle is highly symmetric (8-way symmetry). If you calculate one point $(x, y)$, you can determine 7 other points.
  • Midpoint Circle Algorithm: Uses a decision parameter to find the closest pixel to the true circle boundary. It avoids square root calculations.
  • Clipping: Definition: The process of determining which parts of an object are visible (inside a window) and removing the parts that are outside.
  • Cohen-Sutherland Line Clipping: Uses a 4-bit region code (TBRL - Top, Bottom, Right, Left) to identify position relative to the window.
    • Region Codes: Bit 1 (Left), Bit 2 (Right), Bit 3 (Bottom), Bit 4 (Top).
    • Trivial Acceptance: Both endpoints have code 0000.
    • Trivial Rejection: Bitwise AND of endpoint codes is not 0000.
  • Midpoint Subdivision Algorithm: A divide-and-conquer line clipping algorithm.
    • It bisects the line segment until the visible portion is isolated.
    • Avoids complex intersection calculations by using simple midpoint formulas ($x_m = \frac{x_1+x_2}{2}, y_m = \frac{y_1+y_2}{2}$).
  • Sutherland-Hodgeman Polygon Clipping: Algorithms to clip polygons against the edges of a clipping window. It processes the polygon vertices against each clip edge (Left, Right, Top, Bottom) sequentially.
    • Input: Vertices of polygon. Output: New set of vertices defining clipped polygon.
    • Rules:
      1. Both in: Save 2nd vertex.
      2. In to Out: Save Intersection.
      3. Out to Out: Save None.
      4. Out to In: Save Intersection and 2nd vertex.
  • Weiler-Atherton Algorithm: Used for concave polygons (Sutherland-Hodgeman fails for concave/re-entrant polygons).

Geometrical Transformations (2D)

  • Concept: Methods to modify the shape, size, or position of an object.
  • Translation: Moving an object. Equations: $x' = x + t_x$, $y' = y + t_y$.
    • Matrix: $\begin{bmatrix} 1 & 0 & t_x \\ 0 & 1 & t_y \\ 0 & 0 & 1 \end{bmatrix}$
  • Rotation: Rotating by angle $\theta$. Positive $\theta$ is Counter-Clockwise (CCW).
    • Equation: $x' = x \cos\theta - y \sin\theta$, $y' = x \sin\theta + y \cos\theta$.
    • Matrix: $\begin{bmatrix} \cos\theta & -\sin\theta & 0 \\ \sin\theta & \cos\theta & 0 \\ 0 & 0 & 1 \end{bmatrix}$
  • Scaling: Changing size coordinates. $x' = x \cdot s_x$, $y' = y \cdot s_y$.
    • Matrix: $\begin{bmatrix} s_x & 0 & 0 \\ 0 & s_y & 0 \\ 0 & 0 & 1 \end{bmatrix}$
  • Reflection: Mirror image about an axis ($x$-axis, $y$-axis, $y=x$).
    • About X-axis ($y \to -y$): $\begin{bmatrix} 1 & 0 & 0 \\ 0 & -1 & 0 \\ 0 & 0 & 1 \end{bmatrix}$
    • About Y-axis ($x \to -x$): $\begin{bmatrix} -1 & 0 & 0 \\ 0 & 1 & 0 \\ 0 & 0 & 1 \end{bmatrix}$
    • About Line $y=x$ (Swap $x,y$): $\begin{bmatrix} 0 & 1 & 0 \\ 1 & 0 & 0 \\ 0 & 0 & 1 \end{bmatrix}$
    • About Line $y=-x$: $\begin{bmatrix} 0 & -1 & 0 \\ -1 & 0 & 0 \\ 0 & 0 & 1 \end{bmatrix}$
  • Shearing: Slanting the object. X-shear: $x' = x + sh_x \cdot y, y' = y$.
  • Homogeneous Coordinates: Representing 2D points as $(x, y, 1)$ to allow matrix multiplication for translation.
  • Composite Transformation: Sequence of transformations e.g., Rotation about a pivot point $(x_f, y_f)$:
    1. Translate pivot to origin ($T(-x_f, -y_f)$).
    2. Rotate about origin ($R(\theta)$).
    3. Translate back ($T(x_f, y_f)$).
  • 3D Transformations: Extension to 3D $(x, y, z, 1)$.
    • Rotation: Can happen about X, Y, or Z axis (more complex matrices).

Curves & Surfaces

  • Polygon Meshes: Collection of vertices, edges, and polygons representing 3D surfaces.
  • Parametric Representation: Surfaces defined by parameters $(u, v)$ instead of $y=f(x)$, allowing complex shapes.
  • Interpolation vs. Approximation:
    • Interpolation: Curve passes through all control points.
    • Approximation: Curve is influenced by control points but doesn't necessarily pass through them (e.g., Bezier, B-Spline).
  • Bezier Curve:
    • Defined by $n+1$ control points. Degree of curve is $n$.
    • Properties:
      • Endpoint Interpolation: Passes through first ($P_0$) and last ($P_n$) points.
      • Convex Hull: The curve lies completely within the convex hull of its control points.
      • Global Control: Moving any single control point changes the entire shape of the curve.
  • B-Spline Curve:
    • Generalization of Bezier curves to overcome its limitations.
    • Properties:
      • Local Control: Moving a control point only affects a specific segment of the curve.
      • Degree Independence: The degree of the curve is independent of the number of control points.
  • Continuity:
    • $G^0$: Curves touch (Position continuity).
    • $G^1$: Same tangent direction (Slope continuity).
    • $C^1$: Same tangent direction and magnitude (Parametric continuity).
  • Comparison:
    • Bezier: Simple, Global control, Degree depends on points ($n$).
    • B-Spline: Complex, Local control, Degree independent of points, can represent conics.
  • 3D Projections: Converting 3D objects to 2D screen.
    • Parallel Projection: Preserves relative proportions (e.g., Orthographic). Good for engineering.
    • Perspective Projection: Objects farther away appear smaller. Realistic. Parallel lines meet at a Vanishing Point.
  • Hidden Surface Removal: Determining visible surfaces.
    • Back-Face Culling: Hides polygons facing away.
    • Z-Buffer (Depth Buffer): Pixel-based depth comparison.

Computer Animation

  • Definition: Creating fluid motion by displaying a sequence of still images (frames) quickly.
  • Frame Rate: Smooth motion typically requires 24-30 fps.
  • Key Concepts:
    • Keyframing: Defining start and end states; computer interpolates intermediates.
    • Tweening (In-betweening): Generating intermediate frames between keyframes.
  • Morphing: Transforming one image into another. Steps: Warping (Shape distortion) & Cross-dissolving (Color fading).
  • Motion Specifications:
    • Direct Motion: Explicitly setting rotation/translation values.
    • Goal-Directed: Defining the final state (e.g., "Walk to X").
  • 3D Animation Process: Modeling $\to$ Layout/Animation $\to$ Rendering.
  • Multimedia Standards & File Formats:
    • TIFF (Tagged Image File Format): Lossless, high quality, used in print/publishing.
    • JPEG (Joint Photographic Experts Group): Lossy compression, best for photographs. ($8\times8$ DCT).
    • GIF (Graphics Interchange Format): 8-bit color (256 colors), supports simple animation, LZW compression.
    • MPEG (Moving Picture Experts Group): Standard for video compression.
    • MIDI (Musical Instrument Digital Interface): Communication protocol for digital instruments (stores notes/velocity, not sound waves). Small file size.
  • Components: Text, Audio, Graphics, Video, Animation.
  • Authoring Tools: Software to create multimedia applications.
    • Types: Card/Page-based (HyperCard), Icon-based (Authorware), Time-based (Flash/Animate).
  • JPEG Compression Steps: Image Division ($8\times8$ blocks) $\to$ DCT $\to$ Quantization $\to$ Zig-Zag Scan $\to$ Entropy Encoding.
